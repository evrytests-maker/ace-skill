# Архитектура запроса (Prompt Architecture)

## Базовый порядок промта (дефолтный пресет)

Запрос к модели — это «слоеный пирог», состоящий из набора промтов. Порядок отправки определяется пресетом.

**Порядок слоев (от наименьшего влияния к наибольшему):**

```
+---------------------------------------------------------+
|  1. Persona Description (Описание Персоны)              |
|     - Промт пользователя, задающий роль модели           |
+---------------------------------------------------------+
|  2. World Info (before) — ДО описания бота              |
|     - = ↑Перс = меньшее влияние                        |
+---------------------------------------------------------+
|  3. Character Description (Описание персонажа)          |
|     - Описание бота                                      |
+---------------------------------------------------------+
|  4. World Info (after) — ПОСЛЕ описания персонажа       |
|     - = ↓Перс = большее влияние                        |
+---------------------------------------------------------+
|  5. Example Messages (Примеры сообщений)                |
|     - Образцы сообщений бота                             |
+---------------------------------------------------------+
|  6. Author's Note (Авторские заметки)                   |
|     - Заметки автора                                     |
+---------------------------------------------------------+
|  7. Chat History (История чата)                         |
|     - Последнее сообщение пользователя                   |
|     - САМЫЙ сильный уровень влияния                    |
+---------------------------------------------------------+
```

---

## Важнейшее правило влияния

> **Чем дальше от конца набора промтов — тем МЕНЬШЕ промт влияет на ответ модели. Чем ближе к концу — тем БОЛЬШЕ влияние.**

World Info повторяется ДВА раза в пресете — это ключевая особенность лорбука:

| Позиция | Обозначение | Влияние |
|---------|-------------|---------|
| World Info (before) — ДО описания бота | `↑Перс` | Меньшее влияние |
| World Info (after) — ПОСЛЕ описания бота | `↓Перс` | Большее влияние |

**Записи с after_char повлияют на ответ бота БОЛЬШЕ, чем записи с before_char.**

## Позиции вставки (Insertion Positions)

| Позиция | Код | Название | Назначение |
|---------|-----|----------|------------|
| 0 | `↑Перс.` | Перед описанием персонажа | Для описания того, что имеет смысл расписать **ДО** объявления персонажа: основы мира, базовые настройки сеттинга, общие факты, условно то «во что» мы будем помещать бота. |
| 1 | `↓Перс.` | После описания персонажа | Для описания особенностей персонажа: уточнения его черт. Логичный порядок: сначала описание, потом уточнение. **Наиболее часто используемая позиция.** |
| 2 | `↑ПС` | Перед примерами сообщений | Для языковых инструкций, крылатых фраз сеттинга, стилистических указаний. |
| 3 | `↓ПС` | После примеров сообщений | По факту находится перед Историей чата (чаще всего). Для предыстории нынешней ролки, саммери скрытых сообщений. |
| 4 | `На глуб.` | В Историю чата, начиная с конца | **САМОЕ СИЛЬНОЕ влияние** на ответ модели. Для актуальной информации «здесь и сейчас»: временные эффекты, текущие события. |
| — | `↑АЗ/↓АЗ` | Перед/после авторских заметок | Плавающая позиция, зависит от положения Авторских заметок. |
| — | `➡️Outlet` | Ручная вставка | Записи вставляются через макрос `{{outlet::НазваниеГруппы}}` в пресет. |

---

### Подробное описание каждой позиции

#### ↑Перс. (position=0) — Перед описанием персонажа

- Для описания того, что имеет смысл расписать **ДО** объявления персонажа
- Основы мира
- Базовые настройки сеттинга
- Общие факты
- Условно то «во что» мы будем помещать бота

#### ↓Перс. (position=1) — После описания персонажа

- Для описания особенностей персонажа: уточнения его черт
- Логичный порядок: сначала описание, потом уточнение
- **Наиболее часто используемая позиция**

#### ↑ПС (position=2) — Перед примерами сообщений

- Для языковых инструкций
- Крылатых фраз сеттинга
- Стилистических указаний

#### ↓ПС (position=3) — После примеров сообщений

- По факту находится перед Историей чата (чаще всего)
- Для предыстории нынешней ролки
- Для саммери скрытых сообщений

#### На глуб. (position=4) — В Историю чата, начиная с конца

- **САМОЕ СИЛЬНОЕ влияние на ответ модели**
- Для актуальной информации «здесь и сейчас»
- Временные эффекты
- Текущие события
- Роли: ⚙️ = Система, 🙍 = Пользователь, 🤖 = Ассистент
- Влияние роли зависит от провайдера и модели

#### ↑АЗ/↓АЗ — Перед/после авторских заметок

- Плавающая позиция, зависит от положения Авторских заметок
- Лично автор обычно ее не использует

#### ➡️Outlet — Ручная вставка

- Записи вставляются через макрос `{{outlet::НазваниеГруппы}}` в пресет
- Нужно заполнить поле **Outlet Name** (без пробелов)
- Несколько записей с одинаковым Outlet Name объединяются в группу
- **Преимущество:** ручной контроль позиции через пресет

## Приоритет (Priority / Order)

**Приоритет** — числовая настройка, отвечающая за порядок вставки записей с **ОДИНАКОВЫМ** Положением вставки в промт.

### Правило приоритета

> **Чем больше число — тем НИЖЕ в промте. Чем ниже в промте — тем БЛИЖЕ к чату — тем БОЛЬШЕ влияние.**

### Пример

Две записи с позицией `↓Перс.`:
- У одной `order=100`
- У другой `order=200`

| Order | Порядок вставки | Положение относительно чата | Влияние |
|-------|----------------|----------------------------|---------|
| 100 | Первая (выше) | Дальше от чата | Меньше |
| 200 | Вторая (ниже) | Ближе к чату | **Больше** |

Запись с `order=200` вставится ниже (ближе к чату) и будет сильнее влиять на ответ модели.

---

## Влияние длины чата на промт

Каждый ответ пользователя отсылает модели огромный промт, в который входит и история чата.

> **Чем больше сообщений в чате — тем длиннее промт.**

## Визуальная схема: Иерархия влияния

```
ВЛИЯНИЕ:    МАЛОЕ ◄───────────────────────────────► СИЛЬНОЕ
             │                                              │
             ▼                                              ▼
+------------------------------------------------------------------+
|  ↑Перс.                                                          |
|  (position=0)      ← НАИМЕНЬШЕЕ влияние                          |
|  ↑ПС                                                             |
|  (position=2)                                                    |
|  Описание персонажа                                              |
|  ↓Перс.                                                          |
|  (position=1)                                                    |
|  Примеры сообщений                                               |
|  ↓ПС                                                             |
|  (position=3)                                                    |
|  Авторские заметки                                               |
|  На глуб. (position=4) / ↑АЗ / ↓АЗ / ➡️Outlet                    |
|  История чата        ← САМОЕ СИЛЬНОЕ влияние                     |
+------------------------------------------------------------------+
```

### Упрощенная шкала влияния

```
СЛАБОЕ ВЛИЯНИЕ                                    СИЛЬНОЕ ВЛИЯНИЕ
    │                                                    │
    ▼                                                    ▼
+-------------------------------------------------------------+
|  ↑Перс  →  Описание персонажа  →  ↓Перс  →  История чата   |
|   (0)          (встроен)           (1)        (САМОЕ СИЛЬНОЕ) |
+-------------------------------------------------------------+
       Меньше влияние    ◄─────►    Больше влияние
```

## Ключевые правила (кратко)

1. **Чем ближе к концу промта → тем больше влияние**
2. **World Info вставляется дважды**: `↑Перс` (слабее) и `↓Перс` (сильнее)
3. **Записи с `after_char` влияют больше**, чем записи с `before_char`
4. **Приоритет (order)**: чем больше число — тем ниже в промте — тем ближе к чату — тем сильнее влияние
5. **На глуб. (position=4)** — максимальное влияние, для актуальной «здесь и сейчас» информации
6. **Чем больше сообщений в чате — тем длиннее промт**
